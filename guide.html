Server hosting guide

This guide will guide you through making you're own self hosted EaglercraftX server, this will be the same way I host my server, there is other ways you can also host it but this is the way I personally suggest.
Please make sure you read every single instruction, unless instructed to not, to make sure this guide will be helpful and not fail or lead to a lost system.

Quick note, I am not responsible for any lost files or broken hardware.

To start off here is everything you will need to follow this guide:

  • A spare computer with atleast 32 Gigabytes of empty storage, 4 Gigabytes of ram, and a decent CPU. For this guide we will be calling this computer, the server computer.
    • I very much suggest that you use a different computer from the one you mainly use, as we want to make sure it has enough resources to run the server, and also not mess with your current setup. If you have any spare laptops or random old pcs just use one of those.
    • In this guide we will be installing Ubuntu Linux Server onto this computer, so as with any new OS installation there is always a chance of losing any files on the computer already so please BACK UP ANY IMPORTANT DATA ON THE SERVER COMPUTER BEFORE FOLLOWING THIS GUIDE!!
    • I'm not entirely sure how well it would perform but you could use a raspberry Pi, I would not use a SD card as its main storage, and instead use a M.2 SSD; As the SD card slow disk speeds, will be a HUGE bottleneck that will slow down your server. You also have to know that some instructsions will be different, and as I don't have a raspberry pi to test it on, they will not be included in this guide.
    • You cannot use your school issued chromebook for this
  • A USB Flash drive with 4 Gigabytes of storage, or a SD Card with 4 Gigabytes of storage and an SD Card to USB adapater, or if the server computer has a dvd drive: a blank DVD that you can write to.
  • A Computer running Windows, Mac*, or Linux, or an android device. For this guide we will be calling this the "Main computer"
    • *: You can use a computer running mac, but as I don't have a mac computer to test this with, there are no instructions for that.
    • You cannot use your school issued chromebook for this either.
  • OPTIONAL A second level domain (like the underlined part in google.com, minecraft.net, or wikipedia.org)
    • Needed if you want to use tunneling and use more than 1 Gigabytes a month, host a client, have a dynmap, or have a website along side you server.
    • CANNOT be a subdomain
    • I'd suggest going to namecheap.com/market/, and entering every single one into your chromebook until you find one thats not blocked.
  • OPTIONAL A subdomain that lets you make A records
    • To use this methord you are REQUIRED to port forward, so it doesn't use tunneling.
    • Can also be a second level domain
    • I'd suggesting getting a free one from freedns.afraid.org

An example of a workable setup, Here for this guides server computer I'll be using a 10 year old sony vaio laptop, my dad used to use it for work but he gave it to me because it was just sitting around unused.
This laptop has a Intel Core i7-4500U CPU @ 1.80GHz, and 8 Gigabytes of ram.
Right beside it is a SD Card to USB adapter, that will be using a micro SD card to SD card adapter.

Personally for all server applications, I will always suggest using Linux to host them, as most server applications don't need a desktop, and operating systems like Windows or Mac waste resources on that and other things that will go unused.
For this guide we will be using Ubuntu Linux Server

In this section you'll create the proxy server, which will manage players joining from EaglercraftX

  1. Make a directory that will hold the proxy server

  4. Create a plugins folder
  5. Change directory to the plugins folder
  6. Download the EaglercraftXBungee jar file
  7. Go up a directory
  8. Make a testing .sh file
  9. Allow that .sh file to be ran
  10. Run the .sh file to create all the files waterfall uses
  11. Once it finishes just run the command
  12. Open up the Nano text editor on the file config.yml
    • If you ever need to find a specific thing, you can do "CTRL+W" to search for it
    • Change ip_forward to true
    • Under "servers:" set the address to localhost:25564
    • Under "listeners:" set query_port to 25565
    • Right beneathe that change the motd to whatever you want
      • The motd is the little text under the server name in the minecraft multiplayer menu
      • This text will only be visible to people on minecraft java edition.

    • right beneathe that Change the max_players to whatever number you want

Section 4: Systemd services for the Minecraft server

In this section, we will be making Systemd services for the minecraft server, this will allow you to run the server in a much easier way, and allow the server to auto start whenever you turn on the server computer.

  1. Detatch from the ser-mc user by clicking "CTRL+A+D"
  2. Open up the Nano text editor on a new file called "eagP.service" in the "/etc/systemd/system/" directory
    • Notice how we needed the sudo part this time, thats because the /etc/systemd/system directory is a protected folder, and can only be accesed by root/superuser.
  3. Paste this into the file 
    [Unit]
Description=Minecraft eaglercraft waterfall proxy
After=network.target

[Service]
WorkingDirectory=/home/ser-mc/eagProxy/
User=ser-mc
Group=ser-mc
Restart=on-failure
TimeoutStopSec=10
ExecStart=screen -DmS eagP java -Xmx512M -XX:+UseG1GC -XX:G1HeapRegionSize=4M -XX:+UnlockExperimentalVMOptions -XX:+ParallelRefProcEnabled -XX:+AlwaysPreTouch -XX:MaxInlineLevel=15 -jar waterfall.jar
ExecStop=sleep 5
ExecStop=screen -p 0 -S eagP -X eval 'stuff "stop"\015'

[Install]
WantedBy=multi-user.target
  4. When you are done do "CTRL+O" and enter to save, and "CTRL+X" to leave the editor.
  5. Open up a Nano text editor session on a new file called "[email protected]" in the "/etc/systemd/system/" directory
  6. Paste this into the file 
    [Unit]

[Service]

Section 5: Setting up others being able to join your server.

Now that we have actually created the server, we need to make it so people using different internet can connect to your server. There are three methods I suggest.

The first method I suggest is tunneling with cloudflare. I suggest tunneling through cloudflare over the other options because

  • It's tunneled so it doesn't leak your IP Address.
  • You can use any domain you have, so if you have a domain thats not blocked you can use it.
  • It allows you to create subdomains for your domain that let you tunnel other things like a EaglercraftX client, dynmap, or even a website.
  • All this for the low price of you needing to own your own domain, and thats it.

Cloudflare guide

  1. Start by going to the cloudflare website and creating an account.
  2. Once you've created an account, you'll want to go to your cloudflare dashboard and opent eh "Websites" tab.
  3. Input your domain name in the textbox
  4. Scroll to the bottom and select "Free", then continue.
  5. Follow the instructions in the page, including disabling DNSSEC on your domain and adding the cloudflare name servers
    • On namecheap to add your cloudflare name servers, go to your namecheap dashboard.
    • Click "Manage" on your purchased domain.
    • Under "NAMESERVERS" set it to "Custom DNS" and add both assigned cloudflare name servers that you were given in the cloudflare page.
    • Run the command below on the server computer, you can use SSH to do this.
  6. Once the name servers are added, click "Check nameservers now" and wait for the email saying that your domain was accepted.
  7. Once it finnally says your nameservers were accepted, go to the "Zero trust" tab on cloudflare and set that up for your cloudflare account.
  8. On the Zero Trust dash board, go to the "Networks" tab, and select "Create a tunnel"
  9. Select "Cloudflared" and hit next.
  10. Name the tunnel some descriptive thing for your server computer, and hit Save tunnel.
  11. Set the operating system to Debian, and the architecture to whatever architecture your server computer is, it is likely 64-bit.
  12. Once you run the command switch to the "Public Hostname" tab, and select "Add a punlic hostname"
  13. Set the domain to the domain you added before, and i'd suggest setting the subdomain to something like "eag" or "ser"
  14. Set the type to HTTP, and the URL to "localhost:8081"
  15. Select Save hostname.
  16. Back on the server computer start the cloudflared systemd service.
    • If it doesn't let you exit, and says something like "lines 1-20" just hit the q key.
  17. If you want you can make this auto start by enabling the cloudflared systemd service.
  18. Open up eaglercraft and join the server using the link "wss://cloudflare.domain" with cloudflare.domain replaced with the link you just tunneled using cloudflare.

If you can't pay for an unblocked domain you can use Ngrok as a option. Theres just some things you should know about ngrok

  • ngrok is good because its tunneled, so it doesn't leak your IP Address.
  • With the free tier you can only use the ngrok domain, which is thankfully unblocked.
  • You can only tunnel one thing, which will be your server.
  • After a while of using it, they will start enforcing a 1 GB a month bandwith limit.
  • It's understandable, after all this service is 100% Free!

  9. Start the service
  10. If you want you can also make it start whenever the server computer turns on
  11. Open up EaglercraftX and join the server using "wss://yourNgrokDomain.ngrok-free.app" but replace the yourNgrokDomain.ngrok-free.app with whatever you used before.

IMPORTANT! Because you can't also host a client with ngrok, you will have to download a client file and launch it using lagweird.codehs.me/file.html

This last method uses a little software called Caddy, which lets you serve https data to your domains. Some things to know about Caddy

  • Unlike the other options this is not tunneled, so if you share the domain caddy is connected to, your sorta sharing your IP
  • Instead of using tunneling, this method requires you to port forward!
  • That being said because it isn't tunneled it isn't reliant on any other services, aside from the domain.
  • You can use caddy for a Server, EaglercraftX Client, dynmap, and even a website!
  • All this requires is owning a domain of some kind that lets you make A records, so even a free subdomain from freedns.afraid.org will work.
    Making this method essentially 100% free!

Caddy guide

  1. Port forward port 8081 of the server computer on your router as the Internal and external port as both UDP and TCP
    • It's different for every router so I can't put insturctions here.
    • If you need an internal ip you can do this command on the server computer to find it
  2. Make an A record on your domain pointing to "yourPublicIpAddress:8081" with yourPublicIpAddress replaced with your public ip address
    • You can find your public ip address by visiting whatsmyip.org
    • I'm not 100% sure that this is how you do this part as I cannot test this, so if by the end of this section it works or you find a way that works, please let me know so I can include your working method.
  3. Install caddy by running these commands on the server computer using SSH
  4. Open up a nano session on your caddy file
  5. Change all the text to match This 
    # The Caddyfile is an easy way to configure your Caddy web server.
#
# Unless the file starts with a global options block, the first
# uncommented line is always the address of your site.
#
# To use your own domain name (with automatic HTTPS), first make
# sure your domain's A/AAAA DNS records are properly pointed to
# this machine's public IP, then replace ":80" below with your
# domain name.

example.com {
        # Set this path to your site's directory.
        # root * /usr/share/caddy

        # Enable the static file server.
        # file_server

        # Another common task is to set up a reverse proxy:
        reverse_proxy localhost:8081

        # Or serve a PHP site through php-fpm:
        # php_fastcgi localhost:9000
}

# Refer to the Caddy docs for more information:
# https://caddyserver.com/docs/caddyfile
    • Replace "example.com" with whatever domain you have.
  6. Restart the caddy service
  7. Go to eaglercraftX and join a multiplayer server with the ip "wss://example.com" with example.com replaced with whatever domain you have.

OPTIONAL but still very suggsted Part 3: Extra things to add & know

If you've done all in the guide that lead up to this point you should have a working server, but I suggest you look at this part of the guide where I'll explain some suggsted or good things to know about your server, such as how you can host your own EaglercraftX Client.

Section 1: I suggest adding an auth plugin to your server

If you are running an offline mode server, anyone can set their username to your username, and use your stuff! So to protect it I suggest adding a auth plugin to your server.
The authentication plugin I use is AuthMeReloaded, which is very simple to intstall onto your server.

  1. On the server computer, while connected with SSH, set your user to ser-mc
    • If you're already in ser-mc, ignore this step
  2. Change directory to the minecraft server plugins folder
    • If you named the minecraft server directory something else, you'll replace eagSurvival with whatever you replaced it with
  3. Download AuthMeReloaded legacy jar
  4. Restart the server, and the plugin will be there!

Section 2: Tired of having to use the command line for handling file downloads and file editng?

Don't worry! There's a simpler way! Using Secure File Transfer Protocol or SFTP for short. SFTP Allows you to upload, download, and even edit files on your server computer, and with little setup as openssh already works as a SFTP software!

On windows you can access your servers SFTP server using a program called WinSCP.
Simply download the program, run the program, set the File protocol to SFTP, set the hostname to whatever you use in the ssh command, set the username to ser-mc, and the password to whatever you set the password for ser-mc to. Hit login and you can freely edit any files ser-mc can.

On linux you don't even need to download anything, in your file explorer set the directory to "sftp://ser-mc@hostname" with hostname replaced with whatever you use in the ssh command, and now you can edit any files ser-mc can.

The directory where all your server files will be is in "/home/ser-mc"

Section 3: Hosting a EaglercraftX client.

This section will guide you on hosting an EaglercraftX client on your server computer for you and your players to use on chromebooks so they can join your server.
If you used ngrok you cannot do this part. You need to install nginx for this method, if you don't already have nginx installed just run this command to install it

  1. On the server computer connected with SSH Set your user to root
    • To run sudo commands you need to be in the main account, if you are currently in the ser-mc account do "CTRL+A+D" to detach from it.
  2. Change directory to "/var/www/html"
  3. Download the latest eaglercraftX web build
  4. Install unzip
  5. Unzip the downloaded zip file
  6. Remove the zip file
  7. Let other people access this website using your preferred method
    • If you used cloudflare tunneling, create a new public hostname, with the Type set to "HTTP", the URL set to "localhost:80", and something to differ from the previous hostname you made.
    • If you used caddy
      1. Port forward port 80 of the server computer on your router as the Internal and external port as both UDP and TCP
      2. Make an A record on your domain pointing to "yourPublicIpAddress:80" with yourPublicIpAddress replaced with your public ip address
      3. In the caddyfile add a new section with this in it, replace "example.com" with a new domain IT CANNOT BE THE SAME DOMAIN YOU USED BEFORE 
        example.com {
    reverse_proxy localhost:80
}
  8. Go to the domain and enjoy the client.

Cool thing to note! The "/var/www/html/" directory is everything in your new website. If you wanted you can change the index.html there to your own website, and have it link to the client index.html one way or another. Just tinker around with it if you're interested.
